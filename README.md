# 宣传物料敏感词检索网页

这是一个用 `HTML + CSS + JavaScript` 原生写的本地小工具。  
不需要装框架，也不需要启动复杂环境，适合直接在电脑上打开使用。

## 目录说明
- `index.html`
  网页入口，双击它就能打开页面
- `style.css`
  页面样式
- `script.js`
  检测逻辑、词库读取、历史记录等功能
- `敏感词词库.csv`
  默认词库
- `词库模板.csv`
  上传新词库前可先下载的标准模板

## 怎么打开网页
最简单的方法：

1. 打开这个项目文件夹
2. 找到 `index.html`
3. 直接双击 `index.html`
4. 浏览器打开后就可以开始使用

如果你更想稳一点，也可以：

1. 用 VS Code 打开这个文件夹
2. 安装 `Live Server`
3. 右键 `index.html`
4. 点 `Open with Live Server`

## 怎么使用
1. 打开网页后，左侧输入框里可以直接粘贴文案
2. 点击“开始检测”
3. 页面会自动：
   - 在原文里高亮敏感词
   - 在右侧显示命中词、分类、风险等级、替换建议、备注
   - 计算整体风险等级
4. 如果你要导入新的词库，可以先点击“下载 CSV 模板”
5. 按模板填写后，再点击“上传 CSV 词库”
6. 每次检测记录会自动保存在浏览器本地，刷新页面后还在

## 词库格式要求
上传的新 CSV 词库，请尽量保持下面这 5 列，顺序也保持一致：

```text
word,category,risk_level,replace,note
```

例如：

```text
提分,效果承诺,高,学习进步/能力提升/素养提升,涉及成绩承诺
包过,承诺类,高,提供学习支持,
轻松,夸大宣传,中,更有方法,
```

如果你不想自己从零建表，直接下载页面里的“CSV 模板”再填写会更稳。

## 如果打不开网页，怎么排查
1. 先确认是不是打开了 `index.html`
2. 如果页面样式乱了，检查 `style.css` 和 `script.js` 是否与 `index.html` 在同一目录
3. 如果默认词库读取失败，建议改用 VS Code 的 `Live Server` 打开

## 已实现功能
- 文案输入框，支持粘贴
- 点击“开始检测”执行检测
- 自动匹配敏感词
- 原文高亮显示高风险 / 中风险 / 低风险词
- 右侧展示命中词、分类、风险等级、替换建议、备注
- 展示整体风险等级
- 历史记录保存在 `localStorage`
- 支持上传新的 CSV 词库
- 支持一键替换为推荐表达
- 支持下载 CSV 模板

## 部署说明
仓库已包含 GitHub Pages 自动部署工作流。推送到 `main` 后，会自动尝试部署站点。
